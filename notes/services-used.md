# AWS Services Used

| Service                                             | Use                                                                                                                             | Alternative                                                                                                  |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **EC2** (`t3.micro`)                                | Runs the backend Docker container (FastAPI); single instance with an Elastic IP.                                                | ECS Fargate or Lambda — overkill/more expensive for a personal project at this scale.                        |
| **ECR**                                             | Stores the backend Docker image; EC2 pulls `:latest` on every deploy.                                                           | Docker Hub — ECR keeps everything inside AWS and avoids external auth in the pipeline.                       |
| **RDS PostgreSQL** (`db.t3.micro`, Postgres 16)     | Primary database; stores users, wallets, transactions, budgets, goals.                                                          | Aurora Serverless — cheaper at zero traffic but slower cold starts; SQLite on EC2 — no HA or backups.        |
| **Cognito User Pool**                               | Handles sign-up, sign-in, and issues JWTs; backend validates tokens via JWKS.                                                   | Auth0 / Supabase Auth — Cognito is free up to 50k MAU and stays within AWS.                                  |
| **Lambda**                                          | Pre Sign-Up trigger that auto-confirms new users so email verification is skipped.                                              | Could be inlined as Cognito config, but Lambda is the only supported mechanism for this trigger.             |
| **Amazon Bedrock** (Claude Haiku inference profile) | Powers the "Penny" AI chat assistant; backend calls `bedrock-runtime` with user financial context.                              | OpenAI API — Bedrock keeps inference inside AWS IAM, no external API key needed.                             |
| **S3** (frontend bucket)                            | Hosts the compiled Vite/React static assets; access is locked to CloudFront only via OAI.                                       | Amplify Hosting — S3 + CloudFront is simpler and cheaper for a static SPA.                                   |
| **S3** (pipeline artifacts bucket)                  | Intermediate artifact store for CodePipeline between stages.                                                                    | Required by CodePipeline; no real alternative within the AWS CI/CD toolchain.                                |
| **CloudFront**                                      | CDN serving the frontend; `allow-all` viewer protocol so the browser can call the HTTP backend without mixed-content errors.    | Direct S3 website endpoint — no HTTPS, no CDN caching, no SPA routing support.                               |
| **CodeCommit**                                      | Git repository; push to `main` triggers the pipeline automatically.                                                             | GitHub + CodePipeline GitHub source action — CodeCommit keeps everything inside AWS with no OAuth setup.     |
| **CodePipeline**                                    | Orchestrates Source → BuildBackend → DeployBackend → BuildFrontend on every push.                                               | GitHub Actions — CodePipeline integrates natively with CodeCommit and SSM without extra secrets management.  |
| **CodeBuild**                                       | Three projects: build Docker image, deploy via SSM Run Command, build frontend and sync to S3.                                  | GitHub Actions runners — CodeBuild has native ECR/SSM/S3 IAM access without storing credentials.             |
| **SSM Parameter Store**                             | Stores `DATABASE_URL`, Cognito IDs, and `API_URL`; read by EC2 at runtime and by CodeBuild at frontend build time.              | Secrets Manager — Parameter Store is free for standard parameters; Secrets Manager costs $0.40/secret/month. |
| **SSM Run Command**                                 | Used by CodeBuild to pull the new Docker image and restart the container on EC2 — no SSH or bastion needed.                     | Direct SSH in CI — SSM Run Command is more secure (no open port 22 required in the pipeline).                |
| **IAM**                                             | Roles and policies for EC2 (SSM, ECR, Bedrock), Lambda (basic execution), CodeBuild, and CodePipeline.                          | N/A — required for all AWS service-to-service auth.                                                          |
| **VPC + Subnets + IGW + Route Tables**              | Isolates EC2 and RDS in a private network; RDS is not publicly accessible.                                                      | Default VPC — using a dedicated VPC gives explicit control over SGs and routing.                             |
| **Elastic IP**                                      | Static public IP for the backend EC2 so `API_URL` doesn't change across instance stop/start.                                    | Route 53 A record (supported via optional vars) — EIP is simpler when no domain is needed.                   |
| **Route 53** *(optional)*                           | A record pointing a custom domain to the backend EIP; only provisioned if `route53_zone_id` and `api_domain_name` vars are set. | Direct EIP URL — used only if a vanity domain is desired.                                                    |
